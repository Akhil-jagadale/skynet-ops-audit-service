# Cost Optimization Analysis

## Instance Selection

The application runs on an Amazon EC2 t2.micro instance which is free-tier eligible
and sufficient for lightweight API workloads.

## Containerization Efficiency

Docker containers allow the service to run with minimal resource usage
compared to running multiple virtual machines.

## Minimal Infrastructure

The system uses a single EC2 instance to reduce operational costs
while still demonstrating cloud deployment practices.

## Future Cost Optimization

Future improvements could include:
- Auto Scaling
- Spot Instances
- Serverless architectures
